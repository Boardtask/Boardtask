# Generate a plan

- Include files, logic, method names, and line numvers.
  - To add
  - To update
  - To remove
  
Make plan concise, forsaking grammar for the sake of conciseness.

Break into multiple phases of work.

Provide a way to verify the implementation is complete. eg. for Backend, an API or unit test. For FE, an e2e test. For UI, if given a design, compare a screenshot to real implementation and continue until colour and pixel perfection for the resolution.

Each pase must verify its implementation.

Save as a markdown in project root.
